Hexo博客超贴心安装教程

发表于2022-04-28|更新于2022-04-29
|阅读量:

前言

自己折腾那点事。操作系统 win10 专业版版本号 20H2内部版本 19042.1586 ，MacOS系统和Linux系统下安装基本一样，个别命令行稍有区别自行百度。

1.安装node.js

官网 ： https://nodejs.org

下载链接：https://nodejs.org/dist/v16.15.0/node-v16.15.0-x64.msi

下载链接失效，点击官网下载，选择大众版即可推荐使用迅雷下载速度较快。

node -v > //查看node版本

git-version //查看git版本

3.Git环境变量配置

检查系统变量软件是否自动配置 win+R 打开运行 ==> 输入 sysdm.cpl ==> 打开系统属性*==>* 高级*==>* 点击环境变量 ==> 系统变量找到 Path 点击编辑*==>* 查看是否有 Git/cmd 和 Git/bin 的文件路径，如有则打开cmd输入命令 sh 查看是否可以运行。如无法运行则需找到你所安装Git文件目录下名为 cmd和bin 文件夹，分别复制cmd和bin文件夹的路径粘贴至，编辑新建系统变量中。举例说明，如下图所示。



4.安装Hexo

npm install -g hexo-cli // 使用 npm 全局安装 hexo-cli，如nmp安装成功，则不要安装cnmp。它两是一个东西

npm install -g cnpm --registry=http: //registry.npm.taobao.org//安装淘宝的镜像cnpm 管理器

hexo -v //cmd命令查看hexo版本号，如安装成功则会显示版本号例如hexo-cli: 4.3.0，未安装成功会提示无效命令不显示版本号。

md x:\Hexo //cmd命令新建一个文件夹 x:\Hexo，x是盘符Hexo文件夹名称，可以自定义文件夹的位置和名称。

x: && cd Hexo //cmd命令进入刚才所新建的文件夹，根据上一步自己文件夹位置和名称是什么就改成什么。

注：以下所有操作都必须在自己新建的那个目录下进行 例如 X:\Hexo>

hexo init Hexo //初始化安装hexo，安装完成会在文件夹里显示所有博客文件。

npm install //安装所必须的组件

hexo clean //清理缓存

hexo g //生成博客文件

hexo s //部署本地博客，CTRL+c关闭本地部署

http://localhost:4000 //如部署成功则会出现本地链接复制到游览器即可访问

如有失败请直接删除Hexo文件夹，重新操作安装Hexo

5.部署到Github

npm install --save hexo-deployer-git //下载部署Giehub上所需的插件。

https://github.com //登录自己的Github账号没有的自己注册,官网登录新建一个项目仓库，项目名必须为你的Github用户名+ github.io如下图所示。

git config --global user.name "newnewmingtian" //打开 Git Bash 输入命令设置自己的Git用户名，name后面输入自己Github上的用户名。

git config --global user.email "newmingtian@foxmail.com" //设置自己的Git邮箱/，email后面输入自己Github上的登录邮箱。

记事本打开 Hexo 文件夹内的_config.yml 修改type 为 type: git

新增repo：https://github.com/newnewmingtian/newnewmingtian.github.io.git // repo：后面填写复制自己的GitHub的项目仓库地址。

新增branch: master // branch：后面可自定义。

hexo d //部署推送到Github

hexo clean && hexo g && hexo d //综合命令 清除缓存 生成文件 部署远端，推送到Github仓库。

如有部署失败请直接删除Hexo文件夹内 .deploy_git 和 public 这两个文件夹并重新生成和部署Hexo

完结撒花~!!!
